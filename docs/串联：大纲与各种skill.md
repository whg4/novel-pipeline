# 串联：大纲与各种skill

### 1. 剧情生成原则：【大纲】是唯一绝对核心，【Skill】仅为辅助

*   **严守大纲剧情**：绝不能为了硬套《极简降维Skill》中的规则而去瞎编大纲里没有的情节（例如第4章错误地为了贴合“攻击床笫隐私”而凭空捏造台词）。一切剧情推进、人物动作、场景交互，**必须100%以大纲为准**。
    
*   **不遗漏大纲关键点**：大纲中明确标出的“章末钩子”、“指定台词”（如“把命交给我，信吗？”）必须精准落地，绝不能遗漏。
    

### 2. 叙事视角与行文大忌：杜绝“上帝视角”与“打破第四面墙”

*   **禁忌词汇与大纲术语**：正文中绝对不能出现“全书”、“微观喜好对比”、“降智反应”、“双重物证”这种属于“大纲解析”或“写作技巧”的专业术语或标签词，必须转化为小说里正常的情境描述。
    
*   **拒绝生硬的上帝视角（贴标签）**：不要用旁白去给角色贴标签（如直接写“他在疯狂自欺欺人”），而是要通过角色当下的**具体心理活动、台词和微小动作**（如他死死扣住衣服，在心里默念“我没有错，是她救了我”）来自然展现这一状态。
    
*   **去术语去专业**：用词尽量通俗、极简，去繁冗的修饰，符合大白话的动作流。
    

### 3. 章节连贯性：丝滑的起承转合

*   **开篇无缝衔接**：新一章的开头必须与上一章的结尾**紧密、通顺地咬合**。上一章最后一个角色的动作或反应（如第1章末尾凯尔被威压碾压），必须在下一章开头得到直接的物理或心理承接，不能出现断层。
    
*   **标题格式规范**：每一章开头必须明确使用 `### 第 X 章：章节名` 的标准标题格式。
    

### 4. 交付格式：正文与审查严格分离

*   生成的正文必须干净、纯粹，不夹杂任何解释性废话。
    
*   【逻辑自查】必须放在正文完全结束之后（以分割线隔开），确保您可以**直接复制正文**而不需要手动删减多余内容。
