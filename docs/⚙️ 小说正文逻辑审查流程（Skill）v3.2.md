# ⚙️ 小说正文逻辑审查流程（Skill）v3.2

---

## 审查时机

每写完一章正文后、交付前，必须执行本流程。

---

## 审查步骤

### 第一步：时间线追踪（最重要，极易出错）

逐条列出本章涉及的所有时间节点，不仅要跨章比对，更要同章自检：

| **检查项** | **方法** |
| --- | --- |
| **章内锚点互查** | 提取本章台词带有的时间词，严格核对紧接其后的旁白转场是否完全匹配。 |
| 跨章绝对/相对时间 | “三个月前”“下周二”等，反推具体日期检查是否与前文自洽。 |
| 年龄与倒计时 | N天倒计时逐章严格递减；年龄/周年事件分配不可冲突。 |

**自查提问**：

* 本章时间跨度与台词、上一章结尾是否完美咬合？

---

### 第二步：地点/行程追踪

逐条列出本章角色所在的所有地点和移动轨迹：

| **检查项** | **方法** |
| --- | --- |
| 场景连贯性 | 核心位置转移检验。 |
| 物理状态复用 | 基础场景状态检验。 |

**自查提问**：

* 角色瞬移了吗？同一场景的状态是否被无故重置？

---

### 第三步：道具/物品追踪（核心强化）

列出本章出现的重要道具，检查其前世今生与物理逻辑：

| **检查项** | **方法** |
| --- | --- |
| **名称一致性 (V3.2新增)** | **跨章连续出现的同一物品，指代名词必须绝对统一（前文写“丝带”，后文绝不能写“手帕”；前文写“录音笔”，后文绝不能写“U盘”）。** |
| 获取时间线 | 角色使用该道具时，是否已经在前文明确“拿到了”该道具（防虚空取物）。 |
| 道具状态与数量 | 戒指是否已摘、照片数量、具体金额数字前后必须绝对一致。 |
| 物理常识 | 物品的毁坏或状态变化必须符合其物理材质（纸币不能被“踩碎”）。 |

**自查提问**：

* 角色手里的东西，叫法变了吗？他是什么时候拿到的？毁坏动作符合它的材质吗？

---

### 第四步：人物信息一致性与行为逻辑

| **检查项** | **方法** |
| --- | --- |
| 视角信息传递 | 角色在本章知道什么——严禁开启“上帝视角”。 |
| 已知信息的行为悖论 | 如果角色已得知某物/某事是“假的/废弃的”，其后续言行必须基于此认知。绝不能再发生“明知有假还去用”的降智行为。 |

**自查提问**：

* 角色此刻的反应，符合他当下掌握的信息量吗？

---

### 第五步：情感/节奏检查

| **检查项** | **方法** |
| --- | --- |
| 钩子与倒计时 | 结尾悬念判定；章末是否更新了倒计时格式。 |
| 大纲对齐 | 主线框架核验。 |

---

## 执行方式

写完一章后，在交付前用以下格式做一次自查：

```plaintext
【逻辑自查 - 第X章】
时间线：无冲突 / 有冲突（列明）
地点：无冲突 / 有冲突（列明）
道具：无冲突 / 有冲突（重点排查：获取时间、名称一致性、材质常识）
人物与行为：无冲突 / 有冲突（重点排查：已知信息行为悖论）
情感节奏：章末钩子已设置 / 未设置
倒计时：第N天 ✓ / ✗
```

---

## 🚨 常见致命错误类型（排雷清单）

1. **道具名称与形态突变 (V3.2首要高危)**：同一件信物/道具，跨段落时从“丝带”变成“手帕”，从“项链”变成“戒指”。

2. **提前透支道具**：台词约定“明天把合同给你”，角色却在半小时后的下一场戏直接从怀里掏出了该合同。

3. **已知信息的行为悖论**：反派明知是陷阱，却依然基于错误逻辑进行内讧。

4. **违背常识的材质描写**：高跟鞋“踩碎”纸质美金、“撕裂”钢板等。

5. **章内台词与转场时间割裂**：台词喊“明天见”，旁白接“三天后”。

6. **倒计时跳数字**：上一章剩余6天，本章直接跳到4天。

7. **信息超前泄露**：角色知道了前文根本没有传递给他的机密。
